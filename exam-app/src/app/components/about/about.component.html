About the Exam App

The first step for this app was to create the following components:

  1. Header Component: contains the navigation bar.
  2. Landing Page Component: the first page when a user visits the website.
  3. Login Component: a login page for users to log in.
  4. Signup Component: a Signup page for new users to register.
  5. Result Component: a page where the user can check the results of the exam.
  6. User-profile Component: Where the user can change his profile details.
  7. User-list Component: Where the Admin can check all the users' details including their exam results.

Once all the components were styled and created the next step was to populate the Assessment Component. This was achieved by storing a json file containing the Questions, Options and the Correct-Answers, in the assets folder, and then the question service was used to populate the Assessment Component.

After populating the Assessment component UI, form validation and results display were configured using appropriate functions in the .component.ts files.

The login and Signup components use Template Driven Forms for validation.

The Signup Component uses the api service to create and store new users in the database. On successful creation of new user the user is directly redirected to the user-list component, where all the users' details can be seen.

The API has been set up to run in port 300, and can post new user objects and get all the stored users.
